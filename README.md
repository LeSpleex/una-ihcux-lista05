# ⚡ Heurísticas utilizadas
- `Heurística 01: Visibilidade do Status`: Essa heurística diz que o sistema deve sempre informar ao usuário o que está acontecendo. Utilizei pois da ao usuário uma sensação de andamento do processo, ou seja, melhora a confiança do usuário.
- `Heurística 05: Prevenção de Erros`: Essa heurística diz que o melhor design previne problemas antes mesmo que eles ocorram. Utilizei o try-catch para tratar erros de forma controlada, onde o o bloco try contém o código que pode gerar uma exceção enquanto o catch captura esse erro e define como lidar com ele evitando que o programa pare inesperadamente.
- `Heurística 08: Estética e Design Minimalista`: Essa heurística diz que a interface deve ser limpa, simples e sem excesso de informação. Utilizei para melhorar a leitura do usuário e consequentemente facilitar o entendimento do mesmo.
